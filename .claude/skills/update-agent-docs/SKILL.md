# update-agent-docs

Review `agent-docs/` for staleness against current code, audit against best practices, and apply targeted updates.

## Arguments

- `--check-only` — Run audit only, print report, do not modify files
- `<specific-doc.md>` — Scope to a single agent-doc file (e.g., `cli-layer.md`)
- No arguments — Full audit + interactive scope selection + apply updates

## Workflow

### Phase 1: Audit Against Best Practices

1. Read `agent-docs/README.md` to get the doc index
2. Read the skill's `reference/best-practices.md` for audit rules
3. For each agent-doc file, run structural checks:

   **Package inventory check:**
   ```bash
   ls internal/
   ```
   Compare against packages mentioned in `architecture.md` and `project-structure.md`.
   Flag any package in `internal/` that is not documented.

   **Command inventory check:**
   ```bash
   ls cmd/grafanactl/*/
   ```
   Compare against the command tree in `cli-layer.md`.
   Flag any command group in `cmd/grafanactl/` that is not documented.

   **Pattern count check:**
   Count distinct patterns in `patterns.md`. Compare against actual distinct patterns visible in code (Provider interface, agent mode detection, etc.).
   Flag if new architectural patterns exist but are not cataloged.

   **Config model check:**
   Read the `GrafanaConfig` / `Context` struct shapes in `internal/config/types.go`.
   Compare against the data model diagram in `config-system.md`.
   Flag new fields or structural changes (e.g., `Providers` map).

   **Pipeline count check:**
   Count distinct data flow paths (push, pull, delete, serve, provider-specific).
   Compare against pipelines documented in `data-flows.md`.
   Flag any undocumented pipeline.

   **README index check:**
   Every `.md` file in `agent-docs/` should be listed in `agent-docs/README.md`.
   Flag any orphaned or unlisted doc.

4. Produce audit report: violations grouped by doc file + severity
   - **Stale reference**: A documented path, type, or package no longer exists or has changed
   - **Missing coverage**: A new package, command, pattern, or pipeline is undocumented
   - **Structural issue**: README index is incomplete, metadata is outdated

### Phase 2: Decide Scope

1. If `--check-only` was passed: print the audit report and **stop**
2. If a specific doc was passed (e.g., `/update-agent-docs cli-layer.md`): scope to that file only
3. Otherwise, present the audit findings and ask:

```
Which docs should I update?
- All docs with violations
- Specific files (let me choose)
- Skip (just show the report)
```

### Phase 3: Apply Updates

For each doc in scope, spawn a `runner` sub-agent (haiku model) with:
- The specific agent-doc file path
- The audit violations for that file
- The best-practices rules from `reference/best-practices.md`
- Instructions: **fix violations only**, do not rewrite unaffected sections

After all runners complete:
- Update the `Last updated` line in `agent-docs/README.md`
- Print a summary of what was changed

## Important Notes

- Agent-docs are **high-level architecture references**. Do NOT add low-level details like line numbers or exact function signatures.
- Only flag **structural changes** that shift architecture. Ignore line-level edits, test changes, or formatting differences.
- When updating docs, preserve the existing style and structure. These docs were generated by `/learn-codebase` with consistent formatting.
- The audit checks for what EXISTS in code vs what's DOCUMENTED. It does not check for what's planned or in-progress.
